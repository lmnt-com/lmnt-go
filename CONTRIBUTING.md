## Setting up the environment

This SDK requires [Go](https://go.dev/dl/) 1.23 or newer (see the `go` directive in `go.mod`). To download the dependencies, run:

```sh
$ ./scripts/bootstrap
```

Or run `go mod download` directly.

## Modifying/Adding code

Most of this SDK is generated code. Files carrying a `// Generated by carbonsteel. DO NOT EDIT.` header are regenerated from the LMNT OpenAPI and AsyncAPI specs — edits to them are overwritten on the next regeneration. Hand-written files (this file, `scripts/`, `.github/`, `LICENSE`, `README.md`) are preserved across regenerations.

## Using the repository from source

To use the repository from source, add it to your project with:

```sh
$ go get github.com/lmnt-com/lmnt-go@main
```

## Running tests

Most tests require a [mock server](https://github.com/stoplightio/prism) running against the OpenAPI spec. `./scripts/test` starts one automatically:

```sh
# you will need npm installed
$ ./scripts/test
```

To run the tests against a server you start yourself, set `TEST_API_BASE_URL` in the environment.

## Linting and formatting

This repository uses `gofmt` and `go vet`.

To lint:

```sh
$ ./scripts/lint
```

To format:

```sh
$ ./scripts/format
```

## Publishing and releases

Releases are managed by [release-please](https://github.com/googleapis/release-please). Merging the release PR it maintains tags a new version; Go module consumers pick it up directly from the Git tag, so there is no separate package-registry publish step.
